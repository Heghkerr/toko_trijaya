================================================================================
                    RANGKUMAN PROGRAM TOKO TRIJAYA
                    Sistem Manajemen Toko Retail Modern
================================================================================

1. INFORMASI UMUM
-----------------
Nama Aplikasi    : TOKO TRIJAYA
Framework        : Laravel 10.x
Bahasa Pemrograman: PHP 8.1+
Database         : MySQL/MariaDB
Tipe Aplikasi    : Progressive Web App (PWA) - Dapat diinstall di mobile/desktop
Platform         : Web-based dengan dukungan offline mode

2. TEKNOLOGI YANG DIGUNAKAN
---------------------------
- Laravel Framework 10.x
- Bootstrap 5.3.0 (UI Framework)
- Bootstrap Icons (Icon Library)
- DomPDF (PDF Generation)
- Laravel PWA (Progressive Web App Support)
- Service Worker (Offline Support)
- IndexedDB (Local Storage untuk offline)
- Simple QR Code (QR Code Generation)
- Guzzle HTTP (API Client untuk WhatsApp)
- Tom Select (Advanced Select Dropdown)
- Carbon (Date/Time Manipulation)

3. FITUR UTAMA APLIKASI
-----------------------

3.1. MANAJEMEN PRODUK
   - Master Data Produk dengan informasi lengkap:
     * Nama produk
     * Tipe/Kategori produk
     * Warna produk
     * Supplier/Pemasok
     * Harga beli (HPP)
     * Harga jual
     * Gambar produk
     * Minimum stock (alert stok rendah)
     * Maximum stock (alert stok berlebih)
   
   - Multi Unit System:
     * Satu produk dapat memiliki multiple unit (contoh: Pcs, Lusin, Karton)
     * Konversi otomatis antar unit (contoh: 1 Lusin = 12 Pcs)
     * Manajemen stok per unit dengan konversi otomatis
   
   - Manajemen Tipe Produk:
     * Kategorisasi produk berdasarkan tipe
   
   - Manajemen Warna Produk:
     * Variasi warna untuk produk yang sama

3.2. MANAJEMEN INVENTORI/STOK
   - Tracking stok real-time per unit produk
   - Stok Opname (Stock Taking):
     * Pencatatan stok fisik
     * Penyesuaian stok berdasarkan hasil opname
   
   - Konversi Stok:
     * Konversi stok antar unit (contoh: Lusin ke Pcs)
     * Tracking history konversi
   
   - Alert System:
     * Notifikasi produk dengan stok rendah (di bawah minimum)
     * Notifikasi produk dengan stok berlebih (di atas maximum)
   
   - History Log Stok:
     * Pencatatan semua perubahan stok
     * Tracking user yang melakukan perubahan

3.3. SISTEM TRANSAKSI PENJUALAN
   - Point of Sale (POS) System:
     * Interface kasir yang user-friendly
     * Pencarian produk cepat
     * Multi-item dalam satu transaksi
     * Diskon per item atau total transaksi
     * Perhitungan kembalian otomatis
   
   - Metode Pembayaran:
     * Cash (Tunai)
     * Card (Kartu Debit/Kredit)
     * QRIS (Quick Response Indonesian Standard)
   
   - Struk/Receipt:
     * Generate struk transaksi
     * Print receipt
     * QR Code pada struk
   
   - Status Transaksi:
     * Pending (Belum dibayar)
     * Paid (Sudah dibayar)
     * Completed (Selesai)
   
   - Manajemen Customer:
     * Pencatatan data pelanggan
     * History transaksi per customer

3.4. MANAJEMEN PEMBELIAN
   - Purchase Order (PO):
     * Pencatatan pembelian dari supplier
     * Multi-item dalam satu pembelian
     * Tracking biaya pengiriman
     * Multiple unit per item
   
   - Purchase Return:
     * Retur pembelian ke supplier
     * Tracking retur pembelian
   
   - Manajemen Supplier:
     * Master data supplier
     * History pembelian per supplier

3.5. SISTEM REFUND/PENGEMBALIAN
   - Refund Penjualan:
     * Pengembalian barang dari customer
     * Partial refund (sebagian item)
     * Full refund (semua item)
     * Otomatis update stok dan kas
     * Tracking refund per transaksi
   
   - History Refund:
     * Daftar semua refund yang pernah dilakukan
     * Detail refund per transaksi

3.6. MANAJEMEN KAS (CASH FLOW)
   - Buku Kas:
     * Pencatatan semua arus kas masuk dan keluar
     * Filter berdasarkan:
       - Tanggal (start date - end date)
       - Akun (Cash/Bank)
       - Tipe (Masuk/Keluar)
       - Sumber (Transaction, Purchase, Add Funds, Refund)
   
   - Saldo Kas:
     * Saldo kas tunai real-time
     * Saldo bank real-time
     * Saldo berdasarkan filter tanggal
   
   - Add Funds:
     * Penambahan kas harian
     * Pencatatan setoran awal kas

3.7. SISTEM LAPORAN
   - Laporan Harian:
     * Laporan Penjualan (Sales Report)
     * Laporan Laba Rugi (Profit & Loss Report)
     * Laporan Pembelian (Purchase Report)
     * Filter berdasarkan:
       - Tanggal (range)
       - Kasir/User
       - Tipe laporan
     * Detail produk yang terjual
     * Breakdown metode pembayaran
     * Perhitungan laba rugi per produk
   
   - X Report:
     * Laporan transaksi hari ini
     * Ringkasan penjualan harian
     * Breakdown metode pembayaran
     * Total diskon
     * Kas awal
   
   - Z Report:
     * Laporan penutupan harian
     * Summary akhir hari
     * Download PDF
   
   - Laporan Bulanan:
     * Download laporan dalam format PDF
     * Laporan Penjualan bulanan
     * Laporan Laba Rugi bulanan
     * Laporan Pembelian bulanan
   
   - Laporan via WhatsApp:
     * Pengiriman laporan harian otomatis via WhatsApp
     * Scheduled daily report (setiap jam 4 sore)
     * Format laporan lengkap dengan:
       - Total penjualan
       - Breakdown metode pembayaran
       - Detail produk terjual
       - Laba rugi
       - Saldo kas
     * Integrasi dengan Fonnte API

3.8. DASHBOARD & ANALYTICS
   - Dashboard Overview:
     * Total transaksi hari ini
     * Total pendapatan hari ini
     * Total diskon hari ini
     * Breakdown metode pembayaran (Cash, Card, QRIS)
     * Pendapatan non-cash
   
   - Produk Terlaris:
     * Top 5 produk terlaris hari ini
     * Total quantity terjual
     * Total revenue per produk
   
   - Alert Stok:
     * Produk dengan stok rendah (low stock)
     * Produk dengan stok berlebih (overstock)
   
   - Sales Chart:
     * Grafik penjualan
     * Visualisasi data penjualan

3.9. MANAJEMEN USER
   - Role-based Access Control:
     * Owner (Full access)
     * Kasir (Limited access)
   
   - User Management:
     * CRUD user (Owner only)
     * Assign role per user
     * Profile management

3.10. PROGRESSIVE WEB APP (PWA)
   - Installable App:
     * Dapat diinstall di mobile dan desktop
     * Icon aplikasi
     * Splash screen
   
   - Offline Support:
     * Service Worker untuk caching
     * IndexedDB untuk local storage
     * Offline mode untuk transaksi
     * Sync data ketika online kembali
   
   - Update Notification:
     * Notifikasi ketika ada update aplikasi
     * Auto-update mechanism

3.11. INTEGRASI WHATSAPP
   - WhatsApp Service:
     * Integrasi dengan Fonnte API
     * Pengiriman laporan otomatis
     * Webhook untuk menerima pesan
   
   - Daily Report via WhatsApp:
     * Scheduled report setiap hari
     * Format laporan lengkap
     * Notifikasi ke owner

6. FITUR KHUSUS
---------------
6.1. Multi Unit System
   - Sistem unit fleksibel dengan konversi otomatis
   - Satu produk bisa dijual dalam berbagai unit
   - Tracking stok dalam base unit (Pcs)

6.2. Real-time Stock Management
   - Update stok otomatis saat transaksi
   - Update stok saat pembelian
   - Update stok saat refund
   - Alert stok rendah/berlebih

6.3. Profit Calculation
   - Perhitungan laba per produk
   - Perhitungan HPP (Harga Pokok Penjualan)
   - Laba rugi per transaksi
   - Laba rugi harian/bulanan

6.4. Offline Capability
   - Transaksi dapat dilakukan offline
   - Data tersimpan di IndexedDB
   - Auto-sync ketika online
